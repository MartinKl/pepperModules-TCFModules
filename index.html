This module will allow converting from TCF to Salt and vice versa

General information

TCFModules is part of the Salt'n'Pepper project. This Pepper module allows the conversion of data provided in the XML based Clarin pipeline format TCF to Salt. The conversion from Salt to TCF will be implemented soon.

Purpose

After converting your TCF data to Salt it can be converted to a whole list of other linguistic data formats, e.g. the RelANNIS format used to visualize linguistic data in the linguistic tool ANNIS. A complete list of data formats Salt can be mapped on can be found here.

Brief documentation

The TCF format provides several layers of linguistic annotations. Each TCF layer is mapped on its own layer in Salt, a graph based data format. These layers contain either nodes (e.g. POS annotation), edges (e.g. dependency annotation) or both (e.g. coreference annotation) together with their annotations. The layer's name is the annotation's namespace at the same time. General layer information contained in TCF attributes is stored as meta annotation on the layer itself. A full documentation can be found here soon.

Flags

The converter can be used with a flag SHRINK_TOKEN_ANNOTATIONS. If this flag is true (default), annotations of single tokens are bound directly to the token node in the Salt graph; spans are only used if two are more tokens are meant to be annotated with a single value. If SHRINK_TOKEN_ANNOTATIONS is false, a span is also build over single tokens. In this case all annotations are bound to spans and the annotation layers only contain span elements.
